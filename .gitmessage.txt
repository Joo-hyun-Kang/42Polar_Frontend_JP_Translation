# <タイプ>: <タイトル>
# 本文
# 末尾
# 必要に応じてコメント行(#)を消して使用。Issueトラッキング用
# Issues: #Issue番号
# Resolves: #Issue番号
# See also: #Issue番号
# ------------------
# <タイプ> リスト
#   feat    : 機能 (新機能)
#   fix     : バグ (バグ修正)
#   design  : CSSなどのユーザーUIデザインの変更
#   style   : スタイル (コードの形式、セミコロン追加: ビジネスロジックに変更なし)
#   refactor: リファクタリング
#   comment : 必要なコメントの追加および変更
#   docs    : ドキュメント (ドキュメントの追加、修正、削除)
#   test    : テスト (テストコードの追加、修正、削除: ビジネスロジックに変更なし)
#   chore   : その他の変更 (ビルドスクリプト、パッケージマネージャー設定の修正など)
#   rename  : ファイルまたはフォルダ名の修正や移動作業のみ
#   remove  : ファイルを削除する作業のみ
#   !HOTFIX : 緊急で重大なバグを修正する場合
#   !BREAKING CHANGE : 重大なAPI変更の場合
# ------------------
#     タイトルと本文を1行空けて分ける
#     タイトルの最初の文字を大文字にする
#     タイトルは英語で50文字以内 (日本語は25文字以内)
#     タイトルは命令形で (適用時、このコミットは<コミットメッセージ>します)
#     タイトルの最後にピリオド(.)を付けない
#     本文は英語で72文字ごとに改行 (日本語は36文字ごと)
#     本文は「どうやって」よりも「何を」「なぜ」を説明する
#     本文に複数行のメッセージを書く場合、"-"で区切る
# ------------------
#     末尾はオプションであり、IssueトラッカーIDを記載します
#     末尾は「タイプ: #Issue番号」の形式で使用します
#     複数のIssue番号を記載する場合、コンマで区切ります
#     Issueトラッカーのタイプは以下のいずれかを使用します
#       - Fixes: Issueを修正中 (まだ解決していない場合)
#       - Resolves: Issueを解決した時に使用
#       - Ref: 参考にするIssueがある場合に使用
#       - Related to: 該当コミットに関連するIssue番号 (まだ解決していない場合)
#     例) Fixes: #45 Related to: #34, #23
#-------------------
#   Emoji   説明
#    🎨       コードの形式/構造を改善する時
#    📰       新しいファイルを作成する時
#    📝       小さなコードまたは言語を変更する時
#    🐎       パフォーマンスを向上させる時
#    📚       ドキュメントを書く時
#    🐛       バグを報告する時、@FIXMEコメントタグを挿入
#    🚑       バグを修正する時
#    🐧       Linuxで何かを修正する時
#    🍎       Mac OSで何かを修正する時
#    🏁       Windowsで何かを修正する時
#    🔥       コードまたはファイルを削除する時、@CHANGEDコメントタグと共に
#    🚜       ファイル構造を変更する時 🎨と共に使用
#    🔨       コードをリファクタリングする時
#    ☔️      テストを追加する時
#    🔬       コード範囲を追加する時
#    💚       CIビルドを修正する時
#    🔒       セキュリティを扱う時
#    ⬆️      依存関係をアップグレードする時
#    ⬇️      依存関係をダウングレードする時
#    ⏩       以前のバージョン/ブランチから機能を移行する時
#    ⏪       最新のバージョン/ブランチから機能をバックポートする時
#    👕       linter/strict/deprecation警告を削除する時
#    💄       UI/スタイルを改善する時
#    ♿️     アクセシビリティを向上させる時
#    🚧       WIP (進行中の作業)にコミットする時、@REVIEWコメントタグと共に使用
#    💎       新しいリリース
#    🔖       バージョンタグ
#    🎉       初回コミット
#    🔈       ログを追加する時
#    🔇       ログを減らす時
#    ✨       新しい機能を紹介する時
#    ⚡️      以前のバージョンと互換性のない特徴を導入する時、@CHANGEDコメントタグを使用
#    💡       新しいアイデア、@IDEAコメントタグ
#    🚀       デプロイ/開発作業に関連するすべて
#    🐘       PostgreSQLデータベースに関するもの（マイグレーション、スクリプト、拡張など）
#    🐬       MySQLデータベースに関するもの（マイグレーション、スクリプト、拡張など）
#    🍃       MongoDBデータベースに関するもの（マイグレーション、スクリプト、拡張など）
